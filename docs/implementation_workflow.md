# IMPLEMENTATION WORKFLOW

## Stage 1

Read all documentation.

Summarize understanding.

Identify assumptions.

Wait for approval.

---

## Stage 2

Create implementation roadmap.

Break roadmap into stages.

Wait for approval.

---

## Stage 3

Work on exactly one stage.

Wait for approval.

---

## Stage 4

Work on exactly one page.

Wait for approval.

---

## Stage 5

Implement only approved changes.

---

## Stage 6

Review implementation.

Identify improvements.

Wait for approval.

---

## Stage 7

Commit changes.

Proceed to next stage.

---

# Development Philosophy

Specification
↓
Approval
↓
Implementation
↓
Review
↓
Approval
↓
Commit

Never skip stages.

Never assume approval.

Never implement speculative features.

Never optimize for future requirements unless explicitly requested.
